Anthropic Daily · 2026-04-24

Anthropic 全系 SDK 同步上线 CMA Memory 公测版，支持 Agent 持久化记忆能力；C# SDK 新增 Prompt Caching，CLI 工具增强交互体验。

🧠 SDK 核心功能更新：CMA Memory 公测上线

**Anthropic 宣布 Contextual Memory Augmentation (CMA) Memory 功能进入 public beta 阶段**，并同步更新至所有主流 SDK： - **Python SDK** v0.97.0 - **TypeScript SDK** sdk-v0.91.0 - **Java SDK** v2.27.0 - **Go SDK** v1.38.0 - **Ruby SDK** v1.36.0 - **PHP SDK** v0.17.0 - **C# SDK** v12.17.0 (Bedrock-v0.6.0) **这意味着什么**：CMA Memory 是 Anthropic 在 Agent 持久化记忆能力上的重要布局，支持开发者构建具备长期上下文保持能力的 AI 应用。对于产品经理而言，这意味着可以规划更复杂的 Agent 产品场景，如个人助理、持续对话机器人等，无需用户每次都重复提供背景信息。

💻 C# SDK 新增 Prompt Caching 支持

C# SDK v12.17.0 新增 **Prompt Caching 功能**，通过 `WithCacheControl` 扩展方法集成到 Microsoft.Extensions.AI。 **这意味着什么**：Prompt Caching 可以显著降低重复调用时的延迟和 Token 消耗。适用于有多轮相似上下文的企业级应用，产品经理可据此评估在高频调用场景下的成本优化空间。
anthropic-sdk-csharp Anthropic-v12.17.0

🔧 CLI 工具持续迭代

**anthropic-cli** 密集发布三个版本（v1.3.0 → v1.3.1 → v1.3.2），主要更新包括： - 新增 `--raw-output` / `-r` 选项，支持直接输出非 JSON 格式字符串 - 默认模式切换为 **interactive explore** - 同步支持 CMA Memory public beta **这意味着什么**：CLI 工具的完善降低了开发者的调试成本，interactive explore 模式提升了本地测试体验。产品经理在向工程团队推广 Anthropic API 时，可关注 CLI 作为快速原型验证工具的价值。
anthropic-cli v1.3.0 anthropic-cli v1.3.1 anthropic-cli v1.3.2

🔐 开发者体验优化

- **Go SDK** v1.38.0：为 API errors 新增 `Type()` 方法，便于开发者识别错误类型 - **TypeScript SDK** Bedrock 模块 v0.29.0：为 Mantle client 添加 auth header 支持 - **Ruby / PHP / Java SDK**：修复 API spec 错误并恢复缺失功能 **这意味着什么**：错误处理标准化有助于构建更健壮的 AI 应用。TypeScript Bedrock 集成的改进表明 Anthropic 持续深化 AWS 生态合作。
